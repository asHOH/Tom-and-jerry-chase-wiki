# 网站编辑指南

（最后更新：2026.4.13）  
欢迎来到**猫鼠wiki**，本文介绍如何撰写文章或提供数据。  
目前网站上的大多数信息均可通过账号快速传输、更改，非常方便。  
本板块持续更新中，欢迎关注！  
交流群：**615882730**。

## 注册与登录

**双击**主页底部的“**网站说明**”即可注册或登录。注册时会要求输入：

1. 用户名（登录凭证之一）
2. 昵称（公开展示的名称）
3. 密码（登录凭证之二）

此后即可用用户名和密码直接登录。  
**注意：网站不支持找回账号密码，请妥善保管！**  
登录成功后，右上角会显示一个新的头像图标（若没找到，请刷新页面），然后就能投稿文章或提交编辑记录啦。

- 文章和编辑记录提交后需经管理员审核。未登录时会以匿名形式提交编辑记录，可能导致该记录混杂在其它编辑记录中，不方便审批，建议登录后再提交编辑记录。

## 编辑模式

**点击界面详情页中名称旁的黄色铅笔**进入编辑模式。开启后，主页图标右下角显示黄色铅笔。  
编辑模式支持在线修改绝大多数界面信息（属性/特技/知识卡/技能/克制关系等），但仅在本地生效，需在下方选择“改动记录-提交”，经审核后同步至全站。  
**建议在登录后提交更改，且附加简短的更改信息，以便通过审核！**  
**编辑模式下的更改暂存于浏览器，清理浏览器数据或网站版本更新时可能会丢失部分数据，请及时提交或备份！**

- 更改提交时仅会提交对应界面的更改内容，其余内容仍会暂存。  
  _编辑模式的角色页仍保留了“创建角色”“导出文案”等遗留功能，现已无必要使用。_

### 附：文案的高级语法介绍（20260413版）

在部分文本区域（如角色技能介绍等）允许应用高级语法。（注意：文章暂不支持此类高级语法）  
**所有语法符号均需使用英文符号。正确的语法符号在用户端会隐藏**。  
**尽量不要嵌套使用语法，因为这可能导致显示混乱**。

| 语法           | 格式                                           | 效果                                                                                          | 备注                                                               |
| -------------- | ---------------------------------------------- | --------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| 注释           | `[文本](注释)`                                 | 为文本添加可悬停的注释，注释内容支持嵌套渲染（如 `{}`、`$` 等语法）                           |                                                                    |
| 跳转           | `{项目名}` 或 `《项目名》`                     | 生成站内链接，尝试跳转到对应项目（角色、知识卡、技能等）。技能支持添加类似“1级”“一级”的前缀。 | 支持本名或别名；多个匹配时跳转消歧页                               |
| 定向跳转       | `{项目名(项目类型)}` 或 `《项目名(项目类型)》` | 优先跳转到指定类型的项目，类型如 `知识卡`、`技能`、`特技` 等。                                | 避免歧义，提高链接准确性                                           |
| 伤害           | `{数字}`或`{数字+数字}`                        | 将数字作为 **基础伤害+角色攻击力** 的合计值，生成悬浮提示显示计算明细。                       | 角色攻击力来自当前编辑角色；非角色页视为 0                         |
| 伤害（基础值） | `{数字*}`                                      | 将数字作为 **基础伤害** 值，根据角色攻击力计算应显示值，生成悬浮提示显示计算明细。            | 用于在日后角色基础攻击力发生变化时实时计算伤害                     |
| 破坏伤害       | `{_数字}`                                      | 将数字作为 **基础墙缝伤害+角色墙缝增伤** 的合计值，生成提示                                   | 逻辑同可增伤害，但用于墙缝/破坏场景                                |
| 伤害（带标签） | `{数字,标签1,标签2,...}`                       | 在基础伤害语法上，根据标签修改显示样式和提示内容，标签见下方“伤害标签列表”                    | 支持与“\_”前缀和（或）“\*”后缀连用，其中\*后缀需标于数字后，逗号前 |
| 粗下划线       | `**文本**`                                     | 为文本添加粗体 + 下划线样式                                                                   | 常用于强调                                                         |
| 自定义样式     | `$文本$tailwind样式#`                          | 为文本应用任意 Tailwind CSS 类名                                                              | 例如 `$高亮$text-red-500#` 实现彩色字                              |
| 表达式求值     | `{:表达式}`                                    | 从当前角色对象中读取属性值并显示（如 `{:health}`、`{:skills[0].name}`）                       | 仅支持 `[a-zA-Z0-9_.$]` 的安全路径，不支持运算或函数调用           |

### 伤害标签列表

该内容与[伤害](/mechanics/damage)的分类相关，若此前未了解过，推荐先查阅该机制文章。

| 类别         | 标准名称     | 可用别名（不区分大小写）                                                                                                           | 效果说明                                                                                    |
| ------------ | ------------ | ---------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **来源**     | 有来源       | `有来源`、`有来源伤害`                                                                                                             | 默认标签，无特殊效果                                                                        |
|              | 无来源       | `无来源`、`无来源伤害`、`环境伤害`、`环境`                                                                                         | 显示“无来源”前缀；提示“该伤害无来源，不会触发来源相关效果”；阻止角色增伤                    |
| **计算方式** | 受双方影响   | `受双方影响`、`可增`、`可增型`、`可增型伤害`、`常规`、`常规伤害`                                                                   | 默认计算方式，无前缀；提示受攻击/受击增伤减伤影响                                           |
|              | 只受目标影响 | `只受目标影响`、`受目标影响`、`不受来源影响`、`不增`、`不增型`、`不增型伤害`                                                       | 显示“不受来源影响的”前缀；提示不受攻击增伤/减伤影响，仍受受击影响；可与“无来源”组合隐藏前缀 |
|              | 只受来源影响 | `只受来源影响`、`受来源影响`、`不受目标影响`                                                                                       | 显示“不受目标影响的”前缀；提示不受受击增伤/减伤影响，仍受攻击影响                           |
|              | 固定值       | `不受双方影响`、`不受影响`、`固定值伤害`、`固定型伤害`、`固定`、`固定值`、`固定型`、`不变型`、`不变值`、`不变型伤害`、`不变值伤害` | 显示“固定”前缀；提示不受大多数增伤/减伤影响；阻止角色增伤                                   |
| **电击**     | 电击         | `电击`、`电击伤害`                                                                                                                 | 显示蓝色“电击”前缀；提示可造成感电效果（电击伤害增加、遇水电爆炸）                          |
| **护盾交互** | 可破盾       | `可攻击护盾`、`可击破护盾`、`可破盾`、`可破盾伤害`                                                                                 | 后缀显示“可破盾”；提示会被护盾抵挡并正常消耗层数                                            |
|              | 不破盾       | `不可攻击护盾`、`不可击破护盾`、`不可破盾`、`不可破盾伤害`、`无法攻击护盾`、`无法击破护盾`、`无法破盾`、`无法破盾伤害`             | 后缀显示“不破盾”；提示会被护盾完全抵挡，不消耗层数                                          |
|              | 无视护盾     | `无视护盾`、`无视护盾伤害`、`穿透`、`穿透护盾`、`穿透伤害`、`穿透护盾伤害`、`真实`、`真实伤害`                                     | 后缀显示“无视护盾”；提示无视护盾及绝大多数保护，直接造成伤害                                |
| **致伤**     | 可致伤       | `可致伤`、`可导致受伤`、`会导致受伤`、`会致伤`                                                                                     | 后缀显示“可致伤”；提示成功命中且未被抵挡时使目标进入受伤状态                                |
|              | 不可致伤     | `不可致伤`、`不可导致受伤`、`不会导致受伤`、`不会致伤`                                                                             | 后缀显示“不可致伤”；提示不会使目标受伤                                                      |
| **泡泡交互** | 可攻击泡泡   | `可攻击泡泡`、`可击破泡泡`                                                                                                         | 后缀显示“可攻击泡泡”；提示可以正常攻击泡泡                                                  |
|              | 不可攻击泡泡 | `不可攻击泡泡`、`不可击破泡泡`                                                                                                     | 后缀显示“不可攻击泡泡”；提示无法攻击泡泡                                                    |
| **保护机制** | 无视伤害保护 | `无视保护机制`、`无视保护`、`无视伤害保护机制`、`不受保护机制`、`不受保护`、`不受伤害保护机制`                                     | 后缀显示“无视伤害保护”；提示不会触发猫方伤害保护机制的减伤效果                              |

**使用说明**：

- 标签按类别去重，每个类别仅第一个出现的标签生效。
- 若未指定计算方式标签，系统会根据是否有“无来源”自动选择默认（有“无来源”时默认“只受目标影响”，否则默认“受双方影响”）。
- 可在数字后加 `*` 表示该数字为基础值（如 `{20*,无来源}` 表示基础伤害20，加上角色增伤后显示总和）。
- 标签可自由组合，例如 `{15*,无来源,固定,电击,无视护盾,可致伤}`显示为`15无来源的固定电击伤害（无视护盾，可致伤）`。

## 文章投稿

登录后前往文章板块点击“**新建文章**”即可撰写，审核通过后即可显示。

- **H2/H3/H4**：用于生成目录。请勿将其作为加粗工具，以免目录混淆。
- **角色攻略**：将自动关联至对应角色详情页。

**注意：编辑器无自动保存功能，建议在本地编辑后粘贴。**
